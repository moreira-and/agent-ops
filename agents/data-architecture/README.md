# data-architecture

## Tipo do artefato

discovery

## Finalidade

O diretório `data-architecture/` define perfis de agentes orientados a decisão e desenho arquitetural em engenharia de dados.

Esses agentes devem atuar em atividades como:

- desenho de arquitetura
- definição estrutural
- avaliação de trade-offs
- revisão arquitetural
- organização sistêmica de soluções de dados

A norma de maior precedência continua sendo:

- `../../MANIFEST.md`

---

## Dependências relacionadas

- `../../MANIFEST.md`
- `../README.md`

---

## Quando usar

Consulte `data-architecture/` quando precisar:

- escolher um agente com foco em arquitetura
- orientar decisões estruturais
- avaliar alternativas de desenho
- revisar coerência sistêmica de uma solução
- carregar um perfil com viés de decisão e composição arquitetural

---

## Quando não usar

Não use `data-architecture/` como fonte primária para:

- normas de output
- governança estrutural
- skills operacionais detalhadas
- prompts de tarefa
- execução técnica detalhada de implementação

Consulte, respectivamente:

- `../../rules/`
- `../../governance/`
- `../../skills/`
- `../../prompts/`
- `../data-engineering/`
