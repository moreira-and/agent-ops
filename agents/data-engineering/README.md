# data-engineering

## Tipo do artefato

discovery

## Finalidade

O diretório `data-engineering/` define perfis de agentes orientados à execução de engenharia de dados.

Esses agentes devem atuar em atividades como:

- construção de pipelines
- transformação de dados
- organização de fluxos
- implementação de soluções de dados
- revisão técnica com foco em execução

A norma de maior precedência continua sendo:

- `../../MANIFEST.md`

---

## Dependências relacionadas

- `../../MANIFEST.md`
- `../README.md`

---

## Quando usar

Consulte `data-engineering/` quando precisar:

- escolher um agente executor para tarefas de engenharia de dados
- orientar geração de implementação
- revisar entregas com foco técnico-operacional
- carregar um perfil com viés de execução

---

## Quando não usar

Não use `data-engineering/` como fonte primária para:

- normas de output
- governança estrutural
- skills operacionais detalhadas
- prompts de tarefa
- perfis de decisão arquitetural ampla

Consulte, respectivamente:

- `../../rules/`
- `../../governance/`
- `../../skills/`
- `../../prompts/`
- `../data-architecture/`
